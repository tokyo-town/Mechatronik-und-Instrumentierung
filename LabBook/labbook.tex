\section*{1.1}

Defining $\wk$ as a function of $\sigma$ ensures an accurate representation of the gauss curve in the kernel.
$\wk$ is defined to have the following characteristics:
\begin{itemize}
\item
Odd number of pixels to include the maximum at the center of the kernel.
\item
$\wk$ of at least $\pm 3 \sigma$ around center, so that the kernel covers at least 99 \% of the gauss curve.
\end{itemize}

If $\wk$ is too small, the blur radius is reduced and less blur will occur (see fig.~\ref{fig:kernel_widths}).
The sharp cutoff will also introduce some distortion, as shown in fig.~\ref{fig:kernel_distortion}.

If $\wk$ is too large, many near-zero values will be calculated which increases computation time (see tab.~\ref{tab:kernel_computation})
but does not contribute to the result (see fig.~\ref{fig:kernel_computation}).

\section*{1.2}

Blurring (using a linear filter) smooths out the pixel intensities without changing their mean value.
This is because the linear filter is a moving weighted average of each pixel neighborhood where the sum of coefficients is always 1.
Fig.~\ref{fig:blur_intensities} illustrates this principle for a single row of pixels of a given image.

\bild{images/intensities_comparison.png}{htb}{0.8}{Averaging effect of blur using pixel row 100 of the source image: Pixel intensities with no blur (left) and with Gaussian blur ($\sigma = 5$, $\wk = 31$) (right)}{fig:blur_intensities}


\section*{2.1}

$\lthresh$ determines the lowest possible gradient for a pixel to be considered part of an edge.
A large $\lthresh$ might therefore introduce holes in otherwise continuous edge loops.

$\hthresh$ determines the lowest possible gradient for a continuous edge to be considered valid.
A large $\hthresh$ removes consistently smooth edges (smooth = small gradient) while keeping other edges with at least one sharp section intact.

Fig.~\ref{fig:low_threshold} shows the effect of varying $\lthresh$.
Fig.~\ref{fig:high_threshold} shows the effect of varying $\hthresh$.
See tab.~\ref{tab:thresholdEffects} for a summary of the effects.

\section*{2.2}

Since $\lthresh$ and $\hthresh$ limit gradients, images with different contrast (due to different environments and lighting) will yield different results for the same thresholds.

Adaptive thresholding aims at achieving a largely consistent amount of edges across different environments.
However, fixed thresholding might be more suitable in a stable environment where the amount of detected edges is important and may vary greatly.
For example, we might wish to estimate the number of objects on a table. The more objects, the more edges should be detected.

See fig.~\ref{fig:thresholding_methods} for a comparison of both methods.
$\lthreshauto$ and $\hthreshauto$ have been chosen to achieve a similar amount of edges in the top image.
We can see that in the bottom image, the adaptive method excludes some objects because of the overall edge limit given by $\hthreshauto$,
whereas the fixed method detects objects more reliably in both images.

\bild{images/thresholding_methods.jpg}{htb}{0.8}
{Comparison of fixed and adaptive thresholding methods using images ``table\_object.jpg'' and ``table\_objects.jpg''.
Note that some objects like the glasses or the compass are not detected by the adaptive method.}{fig:thresholding_methods}


\section*{3.1}

Since noise has very high frequency components, it will introduce sharp gradients throughout the whole image which might be detected as edges.
See fig.~\ref{fig:noise_effects} for an illustration of this effect.

\bild{images/noise_effects.jpg}{htb}{0.8}{Effects of noise on edge detection without blur.}{fig:noise_effects}


\section*{3.2}

Because blurring works as a lowpass filter, applying a wide gauss filter (large $\sigma$) will get rid of high frequency noise
and mitigate sharp gradients introduced by it. We can see the lowpass characteristic in fig.~\ref{fig:noise_blur_effects}.
The soft gradients introduced by the low frequency components of the noise can then be sorted out by increasing the hysteresis thresholds.

It is therefore possible to set the parameters such that edges can be detected \textit{as if almost no noise was present (!)}, even in highly noisy images.
However, finer details will be lost due to the blurring (see fig.~\ref{fig:canny_high_noise}).

\bild{images/noise_blur_effects.jpg}{htb}{0.8}{Left: Source image with noise, right: Canny edges with various blur sigmas (no edge hysteresis applied). Note how increasing $\sigma$ reduces the frequency of detected edges.}{fig:noise_blur_effects}
\bild{images/canny_high_noise.jpg}{htb}{0.8}{Top: Source image, bottom: image with high noise ($\sigma = 0.5$). Blur $\sigma = 10$, $\lthresh = 0.1$, $\hthresh = 0.2$ for both images.}{fig:canny_high_noise}
